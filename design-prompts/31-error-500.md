# 500 — Error Boundary — `app/error.tsx`

## What this page is for
The client error boundary shown when a render throws. Reports to Sentry, shows an incident id, offers retry.

## How it looks today
- `grid place-items-center`, `min-h-[100dvh]`, ink-950 with an ember radial glow. `max-w-[600px]`, text-balance.
- Kicker *"Error / 500"*, H1 *"Something broke."*, paragraph *"Something on our end failed. We've been notified — no details needed from you. Try again, or come back in a few."*
- Optional mono *"Incident ID: {digest}"*.
- Buttons: **Try again** (bone, `ArrowClockwise`) + **Back home** (outline, `House`).

Sends `Sentry.captureException` with the digest. Uses `MonoLabel`, Phosphor.

## Redesign goals
- This is the one centered layout that's acceptable (it's a terminal failure state). Keep it calm, in-voice, and honest — no stack traces, a clear retry, and the incident id for support.
